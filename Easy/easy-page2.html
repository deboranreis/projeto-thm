Tarefa 2 | O modelo OSI: Uma visão geral

Qual camada escolheria enviar dados por TCP ou UDP?

Camada 4 - Transporte: A camada de transporte é uma camada muito interessante que desempenha inúmeras funções importantes. Seu primeiro objetivo é escolher o protocolo pelo qual os dados serão transmitidos.

R: 4

Qual camada verifica as informações recebidas para ter certeza de que não foram corrompidas?

Camada 2 - Enlace de Dados: A camada de enlace de dados também desempenha uma função importante ao receber dados, pois verifica as informações recebidas para ter certeza de que não foram corrompidas durante a transmissão, o que pode acontecer quando os dados são transmitidos pela camada 1: a camada física.

R: 2

Em qual camada os dados seriam formatados em preparação para transmissão?

Camada 2 - Enlace de Dados: A camada de enlace de dados concentra-se no endereçamento físico da transmissão.

R: 2

Qual camada transmite e recebe dados?

Camada 1 - Física: A camada física vai até o hardware do computador. É aqui que os pulsos elétricos que compõem a transferência de dados em uma rede são enviados e recebidos. É função da camada física converter os dados binários da transmissão em sinais e transmiti-los pela rede, bem como receber os sinais recebidos e convertê-los novamente em dados binários.

R: 1

Qual camada criptografa, compacta ou transforma os dados iniciais para fornecer um formato padronizado?

Camada 6 - Apresentação: A camada de apresentação traduz os dados em um formato padronizado, bem como lida com qualquer criptografia, compactação ou outras transformações nos dados. Feito isso, os dados são transmitidos para a camada de sessão.

R: 6

Qual camada rastreia as comunicações entre os computadores host e receptor?

Camada 5 - Sessão: Quando a camada de sessão recebe os dados formatados corretamente da camada de apresentação, ela verifica se consegue estabelecer uma conexão com o outro computador na rede. Se não puder, ele retornará um erro e o processo não prosseguirá.

R: 5

Qual camada aceita solicitações de comunicação de aplicativos?

Camada 7 - Aplicação: Funciona quase exclusivamente com aplicativos, fornecendo uma interface para eles utilizarem na transmissão de dados. Quando os dados são fornecidos à camada de aplicação, eles são transmitidos à camada de apresentação.

R: 7

Qual camada lida com o endereçamento lógico?

Camada 3 - Rede: A camada de rede é responsável por localizar o destino da sua solicitação. Por exemplo, a Internet é uma rede enorme; quando você deseja solicitar informações de uma página da web, é a camada de rede que pega o endereço IP da página e descobre o melhor caminho a seguir. Nesta fase estamos trabalhando com o que é chamado de endereçamento lógico (ou seja, endereços IP), que ainda são controlados por software.

R: 3

Ao enviar dados por TCP, como você chamaria os pedaços de dados "pequenos"?

Com um protocolo selecionado, a camada de transporte divide a transmissão em pedaços pequenos (no TCP eles são chamados de segmentos , no UDP eles são chamados de datagramas ), o que facilita a transmissão da mensagem com sucesso.

R: Segments

[Pesquisa] Com qual camada o protocolo FTP se comunica?

O FTP (File Transfer Protocol) é um protocolo da camada de aplicação do TCP/IP que utiliza duas conexões TCP paralelas para transferir um arquivo: uma de controle e outra de dados. O FTP faz parte da camada de aplicação e usa o TCP para fazer as transferências de arquivos.

R: 7

Qual protocolo da camada de transporte seria mais adequado para transmitir um vídeo ao vivo?

O UDP é usado em situações em que a velocidade é mais importante (por exemplo, streaming de vídeo).

R: UDP

Tarefa 3 | Encapsulamento

Como você se referiria aos dados da camada 2 do processo de encapsulamento (com o modelo OSI)?

A Camada 2 organiza os dados em quadros(frames), transmitindo-os sequencialmente e processando os quadros de confirmação mandados de volta pelo receptor.

R: Frames

Como você se referiria aos dados da camada 4 do processo de encapsulamento (com o modelo OSI), se o protocolo UDP fosse selecionado?

Na camada de transporte, os dados encapsulados são chamados de segmento ou datagrama (dependendo se TCP ou UDP foi selecionado como protocolo de transmissão).

R: Datagrams

Que processo um computador executaria em uma mensagem recebida?

Quando a mensagem é recebida pelo segundo computador, ele inverte o processo começando na camada física e avançando até chegar à camada de aplicação, eliminando as informações adicionadas à medida que avança. Isto é conhecido como desencapsulação.

R: De-encapsulation

Qual é a única camada do modelo OSI que adiciona um trailer durante o encapsulamento?

A camada de enlace de dados também adiciona uma peça no final da transmissão, que é usada para verificar se os dados não foram corrompidos na transmissão; isso também tem a vantagem adicional de maior segurança, já que os dados não podem ser interceptados e adulterados sem quebrar o trailer.

R: Data Link

O encapsulamento fornece uma camada extra de segurança (Sim/Não) ?

R: Aye

Tarefa 4 | O modelo TCP/IP

Qual modelo foi introduzido primeiro, OSI ou TCP/IP?

O modelo TCP/IP foi introduzido pelo DoD americano em 1982 para fornecer um padrão – algo que todos os diferentes fabricantes deveriam seguir.

R: TCP/IP

Qual camada do modelo TCP/IP cobre a funcionalidade da camada de Transporte do modelo OSI (Nome Completo)?

O modelo TCP/IP é, em muitos aspectos, muito semelhante ao modelo OSI. É alguns anos mais antigo e serve de base para redes do mundo real. O modelo TCP /IP consiste em quatro camadas: Aplicação, Transporte, Internet e Interface de Rede. Entre eles, cobrem a mesma gama de funções que as sete camadas do modelo OSI.

R: Transporte

Qual camada do modelo TCP/IP cobre a funcionalidade da camada de Sessão do modelo OSI (Nome Completo)?

R: Application

A camada de interface de rede do modelo TCP/IP cobre a funcionalidade de duas camadas no modelo OSI. Essas camadas são Data Link e?.. (Nome Completo)?

R: Physical

Qual camada do modelo TCP/IP lida com a funcionalidade da camada de rede OSI?

R: Internet

Que tipo de protocolo é TCP?

O TCP é um protocolo baseado em conexão. Em outras palavras, antes de enviar qualquer dado via TCP, você deve primeiro formar uma conexão estável entre os dois computadores. O processo de formação dessa conexão é chamado de handshake triplo.

R: Connection-based

Qual é a abreviação de SYN?

SYN (abreviação de sincronizar ), que essencialmente faz o primeiro contato para iniciar o processo de conexão.

R: Synchronise

Qual é a segunda etapa do handshake de três vias?

O servidor responderá então com um pacote contendo o bit SYN, bem como outro bit de "reconhecimento", chamado ACK.

R: Syn/ACK

Qual é o nome abreviado do segmento "Reconhecimento" no handshake triplo?

R: ACK

Tarefa 8 | Escavação

Qual é a abreviação de DNS?

O DNS (Sistema de Nomes de Domínio) é frequentemente comparado a uma lista telefônica. O DNS usa nomes de domínio legíveis por humanos, como www.tryhackme.com, e os mapeia para endereços IP numéricos que podem ser lidos por máquinas.

R: Domain Name System

Qual é o primeiro tipo de servidor DNS que seu computador consultaria ao pesquisar um domínio?

Supondo que o endereço ainda não tenha sido encontrado, seu computador enviará uma solicitação para o que é conhecido como servidor DNS recursivo . Eles serão automaticamente conhecidos pelo roteador da sua rede.

R: Recursive

Que tipo de servidor DNS contém registros específicos para extensões de domínio (ou seja , .com, .co.uk*, etc)*? Use a versão longa do nome.

Os servidores de domínio de nível superior (TLD) são divididos em extensões. Assim, por exemplo, se você estivesse procurando por tryhackme .com , sua solicitação seria redirecionada para um servidor TLD que gerenciava .comdomínios. Se você estivesse procurando bbc .co.uk, sua solicitação seria redirecionada para um servidor TLD que lida com .co.uk domínios.

R: Top-Level Domain

Qual é o primeiro lugar que seu computador procuraria para encontrar o endereço IP de um domínio?

Se nenhum mapeamento tiver sido criado manualmente, o computador verifica o cache DNS local para ver se já possui um endereço IP armazenado para o site; se isso acontecer, ótimo. Caso contrário, passa para a próxima etapa do processo.

R: Local Cache

R: 8.8.4.4
